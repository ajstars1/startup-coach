# Templates — The Belief Ledger

Create as `discovery/LEDGER.md` in your project. IDs are stable forever: strike through answered/killed items with date + answer; never renumber.

```markdown
# Belief Ledger — ⟨venture⟩
Single source of truth. Other docs reference items by ID; movers propose deltas;
the master coach merges. Last updated: ⟨date⟩.

## FACTS (verifiable, with source)
| ID | Item | Source |
|---|---|---|

## ASSUMPTIONS (believed — validation plan + pass/fail + evidence grade)

## MOVEMENT LOG (learning history — append only)
- ⟨date⟩: A⟨n⟩ E⟨x⟩→E⟨y⟩ (⟨mover: interview #/experiment/research⟩) · U⟨n⟩ answered: ⟨…⟩
```

## Evidence grade quick reference

| | What it is | Typical mover |
|---|---|---|
| E0 | Opinion, compliment, AI reasoning | (nothing — counts for nothing) |
| E1 | Told story of past behavior | interview |
| E2 | Observed behavior / artifact | shadowing, logs, usability session |
| E3 | Commitment: time, intros, scheduled next step | interview + follow-through |
| E4 | Transaction: money, repeat purchase | price test, pre-sale, concierge delivery |

Bars: working-validated = E3 ×3 independent · strong = E4 · pricing = E4 only · n=1 = direction, not verdict.
